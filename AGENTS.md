# Transparent Video Object Autoregressive 6D Pose Estimation

## 1. Project Goal
The ultimate goal of this project is to achieve precise, robust, and smooth autoregressive 6D pose estimation for **transparent objects** (e.g., test tubes, beakers, glasses, etc.) in videos. 
For a given video sequence, the inference process follows a sliding-window autoregressive (AR) schedule to ensure temporal consistency:
- **Initial Chunk:** The first 6 frames are processed at once, all acting as target views.
- **Subsequent Chunks:** Each subsequent chunk consists of 6 frames, where the last 3 frames from the previous chunk serve as the **context view** (providing temporal continuity through shared cross-view attention on image features), and the next 3 frames serve as the **target view** for the new pose predictions.

## 2. Architecture & Logic

### 2.1 Core Components
- **Network Architecture (`core/gdrn_modeling/models/GDRN.py`):**
  - **Image Encoder:** Leverages DINOv3 (`dinov3_vits16`) to extract robust patch-level visual features (defaulting to patch size 16), bypassing traditional ResNet/VGGT backbones. DINOv3 provides excellent out-of-the-box semantic part correspondences for transparent objects across frames.
  - **Attention Decoder:** Utilizes Flash Attention and Rotary Position Embeddings (`RoPE2D` for 2D position injection) within transformer blocks (`BlockRope`, `TransformerDecoder`). All N views (context + target) are encoded together and decoded jointly. The decoder alternates between per-view self-attention (`B*N, hw`) and cross-view attention (`B, N*hw`), so context frames directly condition target frame features at the representation level.
  - **View Interaction Module:** When `MODEL.CDPN.VIEW_INTERACTION.ENABLED=True`, a lightweight cross-attention memory block runs before the final pose decode. It builds context-memory tokens from `context roi_feat + xyz cue + rgb + coord2d + mask`, target-query tokens from `target roi_feat + rgb + coord2d + mask`, performs target-to-context attention, and fuses the result back into the target `roi_feat` before the shared `rot_head_net + pnp_net`.
  - **Abs-Only PnP Head:** A single `ConvPnPNet` directly regresses the absolute 6D pose (rotation `allo_rot6d` + translation `centroid_z`) from dense XYZ coordinate maps, region classification, and 2D coordinate features. There is no relative pose branch and no final fusion head. Inference output keys are `abs_rot` / `abs_trans` (aliased to `rot` / `trans`).

- **Training Engine (`core/gdrn_modeling/engine.py`):**
  - Handles forward passes and coordinates multi-task loss computation.
  - Implements dynamic autoregressive chunking during training via `TRAIN_CONTEXT_THEN_TARGET_PROB`. The batch can be treated as either an "all-target" initialization chunk or a "context-then-target" chunk (matching the inference AR window). Context frames flow through the shared decoder and condition target predictions at the feature level.
  - Supervises the abs head only: XYZ coordinate loss, mask loss, region classification loss, point-matching (PM) loss, and direct rotation/translation losses.
  - Features rich Tensorboard debugging, including multi-view fused point clouds, depth maps, and per-view rotation/translation errors.

- **Data Loading (`core/gdrn_modeling/data_loader.py`):**
  - Converts single-instance dictionaries into stacked multi-view batches.
  - Aligns cropped ROIs using DZI (Dynamic Zoom In) patch grid adsorption (`DZI_PATCH_GRID_ADSORPTION`), ensuring the cropped regions perfectly align with the 16x16 ViT patches.
  - Does **not** compute relative rotation/translation matrices (those were removed along with the rel pose branch).

- **Loss Functions (`core/gdrn_modeling/losses/`):**
  - **Pose & Region Losses:** Includes `rot_loss`, `l2_loss`, `pm_loss` (Point Matching), and `loss_ops` for dense coordinate (x,y,z), mask, and region classification.
  - No temporal smoothness losses (TemporalADDSLoss / TemporalSmoothLoss were removed — camera-facing canonicalization makes frame-to-frame R_cf non-smooth even for smooth motion, so temporal rot supervision is incorrect).

- **Autoregressive Inference (`inference_vid_ar.py`):**
  - Implements the chunk-based sliding-window AR logic.
  - Context frame images are processed jointly with target frame images through the shared transformer decoder, providing temporal conditioning at the feature level. When `VIEW_INTERACTION` is enabled, cached context pose / ROI metadata are also consumed by `forward_infer` to crop context RoIs and build a context-side `xyz cue` memory before the final target pose decode. This is still a feature-injection path, not a separate pose head.
  - The network output uses `abs_rot` / `abs_trans` directly. `--pose_source auto` and `--pose_source abs_head` are the only supported choices and are effectively equivalent for the current abs-only checkpoints.
  - **Temporal Smoothing:** Incorporates causal temporal smoothing operations:
    - SLERP for rotation matrices.
    - EMA (Exponential Moving Average) for translation vectors.
    - Adaptive EMA for ROI centers and scales at the result-cache level. The smoothed ROI metadata is saved for analysis/output, but `forward_infer` currently re-extracts target ROIs from masks rather than consuming ROI seeds.

## 4. Current Implementation Status
- **Data Pipeline:** Fully capable of outputting multi-view chunks and aligning bounding boxes to 16x16 patch boundaries (`DZI_PATCH_GRID_ADSORPTION`). Relative pose matrices are no longer computed.
- **B,V,T Tensor Layout Upgrade:** Training/inference code paths now support explicit spatiotemporal layout `input_images(B, V, T, 3, H, W)` in addition to legacy flattened `B,N`. Internally, model forward paths flatten `V*T` for shared token decode while preserving AR scheduling on the time axis and exposing optional `*_bvt` outputs (e.g., `abs_rot_bvt`, `target_view_depth_bvt`) for downstream consumers.
- **Model Architecture:** DINOv3 (ViT-Small) + Transformer Decoder (cross-view attention between context and target frames) + abs-only PnP head. The relative pose branch, `DirectPoseEncoder`, final fusion head, and all context-anchor noise injection machinery have been removed.
  The decoded per-view tokens are bridged into pose-head ROI features by `_encode_view_tokens_to_roi_features`, which runs `roi_decoder -> roi_head -> crop_resize_features -> roi_feat_encoder`; this helper is required by both training/inference code paths that use the VGGT/DINO backbone. If `VIEW_INTERACTION` is enabled, these target ROI features are further enhanced by explicit context-memory attention before the standard `rot_head_net + pnp_net` decode.
  To trade accuracy vs. compute, model depth is now configurable via `MODEL.CDPN.DINO_NUM_BLOCKS` (truncate DINO forward blocks), `MODEL.CDPN.ATTN_DECODER_DEPTH` (cross-view decoder depth), and `MODEL.CDPN.ROI_DECODER_DEPTH` (ROI decoder depth). DINO fine-tuning granularity is additionally configurable via `MODEL.CDPN.DINO_TUNE_LAST_N_BLOCKS`: keep full DINO for forward (`DINO_NUM_BLOCKS=12`) but only unfreeze the last N blocks (plus final norm), with `0` meaning fully frozen. During training, when only last-N blocks are trainable, the frozen DINO prefix is executed under `no_grad` and only the trainable tail participates in autograd, reducing encoder backward/activation overhead while preserving full-depth forward features.
- **Symmetry Handling:**
  - **Continuous symmetry** (e.g., test tubes with rotational axis): Camera-facing canonicalization (`CONT_SYM_GT_MODE="face_camera"`). The canonical spin is chosen so the reference direction `b` (perpendicular to the symmetry axis) points toward the camera. This is a frame-local gauge that is stable and consistent across iterations without needing a history anchor.
  - **Discrete symmetry** (e.g., test tube racks): Hard-coded `closest_pose` selection anchored on the current prediction. This prevents the same image from being assigned contradictory GT symmetry branches across training iterations.
- **Pose/XYZ/Region Supervision Consistency:** The dataloader still reads precomputed raw `xyz_crop`, but training now aligns dense xyz supervision to the same canonicalized pose branch used by pose losses for continuous symmetry. In `GDRN`, after `R_sup = canonicalize(R_raw)`, xyz targets are transformed by `X_sup = S^T (X_raw - o) + o` (`S = R_raw^T R_sup`, `o` is symmetry offset), and this aligned target is used consistently in (1) main xyz loss, (2) geo-prior xyz supervision/debug alignment, (3) depth/mask reprojection loss pose targets (`gt_rot/gt_trans` switched to canonicalized eval pose), and (4) region supervision: `gt_region_supervise` is now recomputed from `gt_xyz_supervise` + `fps_points` + region mask inside `gdrn_loss`, replacing the old raw-xyz dataloader region target for loss.
- **Training Strategy:** The engine dynamically handles context-then-target vs. all-target chunk modes. Context frames participate in shared transformer cross-attention, providing temporal conditioning. With `VIEW_INTERACTION` enabled, training additionally builds context-side memory from GT depth + GT visible mask + (optionally noise-perturbed) context pose, while keeping the final pose output on the same abs-head path. No anchor noise scheduling remains.
- **Training Debugging:** The training loop can auto-log translation-related bad cases to `debug/bad_cases_stats.txt`. Each logged case records the selected view window, target frame names / `scene_im_id`, abs-head translation, GT translation, ROI/camera metadata, mask connected-component stats, a compact loss/vis snapshot, and links to per-iteration pose/depth/mask debug artifacts.
- **Inference Pipeline:** AR inference schedule (e.g., 3 context + 3 target) is functional. The sliding window cleanly isolates context from target. When `VIEW_INTERACTION` is enabled, `forward_infer` reuses cached context pose / ROI seeds to construct context memory, and uses predicted depth + predicted mask together with the cached context pose to backproject a context `xyz cue`. Post-processing includes morphology-based mask cleanups; pose smoothing uses causal SLERP/EMA, and ROI smoothing is currently recorded at the result level rather than fed back into `forward_infer` for target RoI extraction. Use `--pose_source auto` or `--pose_source abs_head`.
  For debugging real-video jitter attribution, AR inference now supports optional per-target tensor dumps via `--save_coor_xyz_npy` (or env `GDRN_SAVE_COOR_XYZ_NPY=1`), saving `coor_z` logits and decoded `xyz` maps as `.npy` files under `pred_ar/coor_z_npy/` and `pred_ar/xyz_npy/` and recording paths in `results.json`. If this debug option is enabled while `cfg.TEST.USE_PNP=False`, `inference_vid_ar.py` now auto-enables `cfg.TEST.USE_PNP=True` at runtime so `coor_x/y/z` tensors are emitted. A second debug option `--save_pnp_decode_debug` (or env `GDRN_SAVE_PNP_DECODE_DEBUG=1`) dumps per-frame raw PnP-head outputs (`pnp_pred_t_raw`, `pnp_pred_rot_raw`, `.npy` files) and decode-side diagnostics (`pnp_decode_debug`) to separate z-head instability from decode-geometry amplification. A third debug option `--save_pnp_input_debug` (or env `GDRN_SAVE_PNP_INPUT_DEBUG=1`) dumps PnP-input-side tensors per target frame (e.g., `coor_feat_base/final`, `region_softmax/atten`, `mask_atten`, `roi_coord_2d`, `prior_xyz/conf/residual`, `xyz_single`) under `pred_ar/pnp_input_npy/`, with paths recorded in `results.json`.
  For train/infer consistency of region semantics, `model_info` now carries per-object `fps_points` in both dataloader and `inference_vid_ar.py` (when available), so region supervision/debug can use the same anchor set.
- **Known Real-Video Failure Mode (IMG_5881, ckpt `68199`):** On a 705-frame real test-tube clip with stable external masks, visible jitter was dominated by `centroid_z` instability rather than ROI/mask jitter: `raw_roi_center` / `raw_scale` stayed smooth while the implied raw z head toggled between lower and higher modes, causing large `tz` jumps and box-height flicker. A separate structural issue also exists in AR inference: if `context_pose_min_conf` gating marks several consecutive target frames invalid, later windows lose all valid context and `prior_conf_64` collapses to zero, so the AR context bank cannot recover on its own. Setting `GDRN_CONTEXT_POSE_MIN_CONF=0` preserves context recoverability, but does **not** eliminate the underlying z jitter and can even worsen aggregate motion, indicating the deeper problem is weak / brittle geometry-confidence and z regression under transparent real-video domain shift, not merely the inference-time confidence threshold.
- **Train/Infer Gap for Context Geometry:** The current VI / geometric-prior training path builds context `xyz cue` from **GT depth + GT visible mask + GT context pose**, whereas inference builds it from **predicted depth + mask observation + cached predicted pose**. This gap can make the context memory and its confidence calibration brittle on real transparent videos even when external masks are supplied.
- **ZFinalRegHead (z refinement via render-and-compare):** Implemented in `core/gdrn_modeling/models/z_final_reg.py`. Activated via `MODEL.CDPN.Z_FINAL_REG_HEAD.ENABLED=True`. Adds a second-stage head after the coarse PnP decode that renders the object model at the coarse pose, compares it to the observed mask, and regresses `log(tz_abs)` directly. Uses a 5-stage anneal training schedule and adds `loss_z_final_reg` + `loss_trans_final_reg` losses. Inference: `forward_infer` uses PLY-loaded, FPS-sampled model points (loaded in `inference_vid_ar.py`) for rendering; refined `pred_trans` flows through the existing EMA temporal smoothing. Debug outputs: `comp_mask`/`obs_mask` PNGs saved to `results_dir/mask_compare/` during inference.
- **Hand-Occlusion Augmentation (`tools/labsim/apply_hand_occ_to_scene.py`):** The labsim hand-occlusion augmenter supports folder-level sampling, treats paired two-hand clips as atomic, and defaults to **not** mixing unrelated left/right single-hand clips. Mixed two-hand composition is opt-in via `--allow-mixed-two-hand`. Its object-aware placement logic is boundary-anchored and now validates candidate placements against the actual per-frame `mask_visib` object masks, enforcing configurable occlusion budgets with both average-sequence and per-frame lower/upper bounds. Minimum and maximum rendered hand size are enforced across **all** frames of a chosen clip via `--min-hand-height-px` and `--max-hand-height-px`, not just the first frame.
- **Online vs Offline Hand-Occlusion Strategy:** The current labsim training pipeline consumes fully materialized scene folders plus per-frame `mask_visib`, `scene_gt_info`, and `xyz_crop` supervision. Because the mapper derives `roi_mask_visib`, `roi_mask_obj`, `roi_xyz`, and ROI crops from those assets, robust hand-occlusion training is currently better served by **offline pre-generation** of augmented scenes. Online hand pasting remains feasible in principle, but only if the dataloader also updates the visible-mask-derived supervision consistently across all frames of a sampled sequence.

## 5. Decision Records (ADR)
- **ADR-001 (Vision Encoder):** Adopted DINOv3 (`dinov3_vits16`) instead of ResNet/VGGT. *Reason:* DINOv3 provides stronger, out-of-the-box semantic part correspondences across frames, which is critical for accurate pose regression.
- **ADR-002 (Autoregressive Context):** Selected a 3-frame context and 3-frame target chunking strategy. *Reason:* 3 frames provide enough temporal velocity/acceleration information without overwhelming memory, while predicting 3 frames at a time speeds up inference compared to pure frame-by-frame AR.
- **ADR-003 (Abs-Only Architecture):** Removed the relative pose branch, final fusion head, and all context-anchor noise injection machinery; retain only the abs-head PnP net. *Reason:* The complexity of the 3-branch system (abs + rel + fusion) with context-dependent canonicalization created interacting failure modes. The abs head is simpler to supervise and debug, and camera-facing canonicalization provides a stable frame-local gauge that doesn't require history. Temporal conditioning is provided at the **feature level** through the shared transformer decoder (cross-view attention), not by injecting predicted poses into the head.
- **ADR-004 (Inference Smoothing):** Applied causal SLERP and EMA at the end of the inference pipeline. *Reason:* While the network predicts per-frame poses independently, explicit low-pass filtering guarantees visual smoothness for rendering purposes.
- **ADR-005 (Camera-Facing Canonicalization for Continuous Symmetry):** For continuous-symmetry objects, the GT spin is canonicalized so the reference direction `b` (chosen orthogonal to the symmetry axis `a` using the most-perpendicular basis vector) faces toward the camera. Formula: `c_obj = -R^T t`, `u_perp = c_obj - (a·c_obj)a`, `d = normalize(u_perp)`, `theta = atan2(a·(b×d), b·d)`, `S_cf = Rodrigues(a, theta)`, `R_cf = R @ S_cf`. Falls back to raw pose if `|u_perp| < CONT_SYM_CANON_DEGENERATE_THRESH` (camera near axis). *Reason:* This is a frame-local gauge — no history or context anchor needed. The same image always maps to the same canonical branch regardless of training iteration, enabling stable abs-head convergence.
- **ADR-006 (No Temporal Rotation Losses):** Removed TemporalADDSLoss and TemporalSmoothLoss. *Reason:* Camera-facing canonicalization makes `R_cf` change with camera motion even for a static object. Computing velocity/acceleration on canonicalized poses would penalize valid predictions as jitter. Temporal smoothness is handled at inference via SLERP/EMA post-processing instead.
- **ADR-007 (Closest-Pose for Discrete Symmetry):** Hard-coded `closest_pose` (prediction-anchored) for discrete symmetry GT selection. *Reason:* The abs head has no context input. Forcing it to track a context-dependent symmetry branch means the same image is assigned different GT branches across iterations, preventing convergence. Prediction-anchoring makes the assignment consistent: each iteration's GT is the candidate closest to the current prediction.
- **ADR-008 (Sequence-Shared Context Anchor Noise):** During context-then-target chunk training, the same noise perturbation is applied to the context anchor for all target views in the chunk. *Reason:* All targets in the chunk share the same injected context anchor; per-target noise would mix different conditioning patterns inside one sequence and be less faithful to inference-time behavior.
- **ADR-009 (Inference Pose Source):** `inference_vid_ar.py` uses `abs_rot`/`abs_trans` directly (`--pose_source auto` or `--pose_source abs_head`). The current CLI only accepts these two abs-head options. *Reason:* The project is now abs-only, so keeping the inference interface focused on the surviving output path reduces ambiguity.
- **ADR-010 (Hand Pair Integrity and Boundary-Anchored Placement):** For labsim hand-occlusion augmentation, a "two-hand" sample must come from a single paired clip; composing unrelated left/right clips is disabled by default. Placement should respect how the source clip enters the frame. *Reason:* Produces substantially more plausible RGB augmentations for training.
- **ADR-011 (Mask-Level Occlusion Budget for Hand Augmentation):** Bounding-box overlap is too coarse for controlling occlusion strength. Candidate hand placements must be evaluated using the resized hand alpha mask against the true source-scene `mask_visib` bitmap for each frame, and accepted only if the per-frame object occlusion ratio stays below the configured budget (currently `<= 20%`). The chosen scale must also satisfy the minimum rendered hand height on every frame of the clip. *Reason:* This keeps augmentation strength realistic and prevents small tail-end hand patches or over-occluding placements from slipping through.
- **ADR-012 (Current Partial-VISOR Regime for Hand Occlusion):** With the current incomplete VISOR subset, requiring every frame in a 12-frame labsim sequence to land inside a strong occlusion band such as `15%-25%` is usually too strict and causes most sequences to fail placement search. A more practical interim regime is: require every frame to have at least a small non-zero occlusion, constrain the sequence-average occlusion into the target band (for example `15%-25%`), and keep per-frame maximum occlusion capped (for example `<= 25%`). Tight per-frame lower bounds can be revisited after downloading a fuller VISOR hand-clip pool.
- **ADR-013 (Prefer Offline-Generated Hand Occlusion for Training):** For the current labsim training stack, hand occlusion should primarily be generated ahead of training rather than pasted on-the-fly inside the dataloader. *Reason:* The dataset/mapper path supervises not just RGB but also visible masks, ROI crops, XYZ targets, region labels, and bbox-derived translation targets over 12-frame sequences. Offline generation keeps these annotations self-consistent, reproducible, and cheap to debug. If online augmentation is added later, it should start as a lightweight, opt-in stage that reuses cached hand clips and recomputes at least the visible-mask-dependent targets consistently, instead of replacing the offline pipeline outright.
- **ADR-015 (Split ConvPnPNet FC into Independent Branches):** Replaced the shared FC bottleneck in `ConvPnPNet` with three independent branches: rotation (`fc_r1→fc_r2→fc_r`), centroid (`fc_c1→fc_c2→fc_centroid`), and z (`fc_z1→fc_z2→fc_z`). All three start from `flatten(8192)` after the shared conv layers. *Reason:* The shared FC bottleneck caused z-gradient interference from rotation and centroid tasks, degrading z regression quality on transparent objects. Independent branches ensure z has its own dedicated feature space with isolated gradients.
- **ADR-016 (ZFinalRegHead Render-and-Compare for Z Refinement):** Added `ZFinalRegHead` in `core/gdrn_modeling/models/z_final_reg.py` as a second-stage z refinement head. It renders a model mask at the coarse predicted pose (`comp_mask`) and compares it against the observed mask (`obs_mask`) to regress `log(tz_abs)` directly. Training uses a 5-stage anneal schedule transitioning from 100% GT render inputs to (10% GT, 50% GT+noise, 40% pred) over warmup + anneal iterations. New losses: `loss_z_final_reg` (SmoothL1 on log-tz) and `loss_trans_final_reg` (L1 on decoded translation) are additive — all coarse losses are preserved. *Reason:* The coarse PnP head lacks explicit size comparison signal. By rendering the model at the predicted pose and comparing it to the observed mask, the head can directly perceive depth errors as size mismatches, providing a strong geometric signal that the coarse head's FC layers cannot learn from per-pixel coordinate maps alone.
- **ADR-017 (Region Supervision Recomputed from Pose-Aligned XYZ):** Region supervision is no longer trusted as a fixed dataloader artifact when continuous-sym canonicalization changes pose/xyz branch. In `gdrn_loss`, after `gt_xyz_supervise` is built, `gt_region_supervise` is recomputed from aligned xyz (`gt_xyz_supervise`) and per-object `fps_points` under the configured region mask; region loss consumes this recomputed label map. If `fps_points` metadata is missing/invalid, logic falls back to the original `gt_region`. `model_info["fps_points"]` is now attached by both dataloader and video inference setup to keep supervision/debug anchor definitions consistent. *Reason:* Prevents raw-xyz region labels from becoming semantically inconsistent with canonicalized xyz/pose targets, which otherwise injects contradictory gradients into the region branch and downstream PnP decode.
- **ADR-018 (Explicit Spatiotemporal Layout B,V,T):** Upgraded data collation, training window sampling, model input handling, and AR inference scheduling to support explicit `B,V,T` tensors (with backward-compatible flatten fallback). Temporal target selection is now expanded across spatial views (`target_time_idx -> target_idx_flat`) so each AR step predicts all views for selected times consistently. *Reason:* The previous `B,N` layout conflated spatial and temporal dimensions, making true multi-camera multi-time training/inference brittle and ambiguous.
